# Contributing

Contributions are welcome. Please read these guidelines before submitting.

## Before You Contribute

- Open an issue to discuss significant changes or new features before starting
  work. This avoids duplicated effort and ensures the change fits the project's
  direction.
- For small bug fixes or documentation improvements, a pull request is fine
  without a prior issue.

## What Belongs in This Repository

This repository is the **open-source macOS desktop pet runtime**. It covers:

- Local pet rendering and animation
- Pet bundle loading and format support
- macOS desktop integration
- Installer and launcher scripts

The following are **out of scope** for this public repository unless explicitly
discussed and agreed upon first:

- AI-agent behavior, memory, voice, or automation
- Cloud sync, accounts, subscriptions, or payment logic
- Premium characters, proprietary assets, or commercial product features
- App-control, filesystem-control, or privileged system integrations
- Plugin systems tied to a commercial marketplace

If you are unsure whether a contribution fits, open an issue first.

## Asset Submissions

**Do not submit** copyrighted sprites, artwork, characters, mascots, sounds,
animations, or trademarks unless:

- You are the original creator and have the right to contribute them, **or**
- They are clearly licensed in a way that permits inclusion in an
  Apache-2.0-licensed project.

If submitting original assets, include the license or a brief statement
confirming you own the rights.

## Contributor Agreement (DCO)

By submitting a contribution (pull request, patch, or otherwise), you certify
that:

1. The contribution was created in whole or in part by you, and you have the
   right to submit it under the Apache License 2.0.
2. You understand and agree that your contribution will be licensed under the
   Apache License 2.0.
3. You have not included material that is covered by a conflicting license or
   that you do not have the right to contribute.

This is a lightweight Developer Certificate of Origin (DCO) statement. No
formal CLA is required at this time.

## Pull Request Process

1. Fork the repository and create a branch for your change.
2. Keep pull requests focused — one logical change per PR.
3. Ensure your changes do not break existing functionality.
4. Update documentation or comments if your change affects behavior.
5. Submit the pull request with a clear description of what you changed and why.

## Code of Conduct

Be respectful. Contributions of all experience levels are welcome.
